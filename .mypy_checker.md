# Python 类型检查指南

## 安装 mypy

```bash
# 安装开发依赖（包括 mypy）
pip install -e ".[dev]"

# 或者只安装 mypy
pip install mypy
```

## 运行类型检查

### 检查整个 src 目录
```bash
mypy src/
```

### 检查特定文件
```bash
mypy src/install.py
mypy src/project.py
```

### 检查特定目录
```bash
mypy src/remote/
```

## 常见类型错误示例

### 1. Path vs str 类型不匹配

**错误代码：**
```python
def func(target: Path):
    os.path.exists(target)  # 错误：Path 对象不能直接用于 os.path.exists
```

**mypy 报错：**
```
error: Argument 1 to "exists" has incompatible type "Path"; expected "str | bytes"
```

**修复方法：**
```python
# 方法1: 转换为字符串
def func(target: Path):
    os.path.exists(str(target))

# 方法2: 使用 Path 的方法
def func(target: Path):
    target.exists()

# 方法3: 接受多种类型
from typing import Union
def func(target: Union[str, Path]):
    os.path.exists(str(target))
```

### 2. Optional 类型未检查

**错误代码：**
```python
def func(value: Optional[str]):
    print(value.upper())  # 错误：value 可能是 None
```

**修复方法：**
```python
def func(value: Optional[str]):
    if value is not None:
        print(value.upper())
```

## mypy 配置说明

在 `pyproject.toml` 中的配置：

```toml
[tool.mypy]
python_version = "3.8"              # 目标 Python 版本
warn_return_any = true              # 警告返回 Any 类型
check_untyped_defs = true          # 检查未标注类型的函数
warn_redundant_casts = true        # 警告冗余的类型转换
warn_unused_ignores = true         # 警告未使用的忽略注释
no_implicit_optional = true        # 不允许隐式 Optional
strict_optional = true             # 严格检查 Optional
show_error_codes = true           # 显示错误代码
pretty = true                     # 美化输出
```

## 忽略特定错误

如果某行确实无法修复，可以添加注释忽略：

```python
result = some_function()  # type: ignore[arg-type]
```

## 推荐工作流

1. 先检查单个文件，了解问题
2. 逐步修复类型错误
3. 最后检查整个项目

```bash
# 步骤 1: 检查单个文件
mypy src/install.py

# 步骤 2: 修复错误后，检查整个目录
mypy src/

# 步骤 3: 将类型检查加入 CI/CD
```
